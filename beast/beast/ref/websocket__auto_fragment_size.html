websocket::auto_fragment_size

Automatic fragmentation size option. 

using auto_fragment_size = implementation-defined;

Sets the maximum size of fragments generated when sending messages on a WebSocket socket. When the automatic fragmentation size is non-zero, messages exceeding the size will be split into multiple frames no larger than the size. This setting does not affect frames send explicitly using write_frame or async_write_frame. The default setting is to fragment messages into 16KB frames.

Remarks

Objects of this type are passed to socket::set_option.

Example

Setting the automatic fragmentation size option: 

...
websocket::stream<ip::tcp::socket> stream(ios);
stream.set_option(auto_fragment_size{8192});
