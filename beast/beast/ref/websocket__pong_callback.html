websocket::pong_callback

Pong callback option. 

using pong_callback = implementation-defined;

Sets the callback to be invoked whenever a pong is received during a call to read, read_frame, async_read, or async_read_frame.

Unlike completion handlers, the callback will be invoked for each received pong during a call to any synchronous or asynchronous read function. The operation is passive, with no associated error code, and triggered by reads.

The signature of the callback must be: 

void callback(
    ping_data const& payload    // Payload of the pong frame
);

If the read operation receiving a pong frame is an asynchronous operation, the callback will be invoked using the same method as that used to invoke the final handler.

Remarks

To remove the pong callback, construct the option with no parameters: set_option(pong_callback{})

Requirements
