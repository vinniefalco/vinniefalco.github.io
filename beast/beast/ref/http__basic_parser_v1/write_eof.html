http::basic_parser_v1::write_eof

Called to indicate the end of file. 

void
write_eof(
    error_code& ec);

HTTP needs to know where the end of the stream is. For example, sometimes servers send responses without Content-Length and expect the client to consume input (for the body) until EOF. Callbacks and errors will still be processed as usual.

Remarks

This is typically called when a socket read returns eof.
