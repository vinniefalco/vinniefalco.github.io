This is a smart pointer that retains shared ownership of an object through a pointer. Memory is managed using the allocation and deallocation functions associated with a completion handler, which is also stored in the object. The object is destroyed and its memory deallocated when one of the following happens:

  • The function invoke is called.
  • The function release_handler is called
  • The last remaining container owning the object is destroyed

Objects of this type are used in the implementation of composed operations. Typically the composed operation's shared state is managed by the handler_ptr and an allocator associated with the final handler is used to create the managed object.

Remarks

The reference count is stored using a 16 bit unsigned integer. Making more than 2^16 copies of one object results in undefined behavior.
