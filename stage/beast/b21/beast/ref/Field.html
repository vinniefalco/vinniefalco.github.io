Field requirements

A Field represents a single HTTP header field/value pair.

In this table:

  • X denotes a type meeting the requirements of Field.
  • a denotes a value of type X.

Table 5. Field requirements

operation

type

semantics, pre/post-conditions

a.name()

boost::string_ref

This function returns a value implicitly convertible to boost::string_ref containing the case-insensitive field name, without leading or trailing white space.

a.value()

boost::string_ref

This function returns a value implicitly convertible to boost::string_ref containing the value for the field. The value is considered canonical if there is no leading or trailing whitespace.
