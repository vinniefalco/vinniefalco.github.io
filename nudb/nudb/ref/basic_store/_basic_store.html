basic_store::~basic_store

Destroy the database. 

~basic_store();

Files are closed, memory is freed, and data that has not been committed is discarded. To ensure that all inserted data is written, it is necessary to call close before destroying the basic_store.

This function ignores errors returned by close; to receive those errors it is necessary to call close before the basic_store is destroyed.
